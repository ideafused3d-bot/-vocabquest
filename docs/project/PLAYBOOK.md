## PB-001 用真實原始碼驗證前端純函式邏輯（2026-07-04 首次使用，遷移收錄）

- 適用情境：要驗證 `common.js`／`student.js` 內不碰 DOM 與網路的純函式（如 `wordStatus()`、`levelInfo()`、`stats()` 的計算口徑）。
- 步驟：寫臨時 Node 腳本，用 `node:vm` 模組載入**真實原始碼檔案**（禁止把邏輯複製貼上重寫一份），注入最小 stub（如空的 `window`），餵合成資料斷言輸出。
- 驗證方式：覆蓋每個狀態分支至少一個樣本（例：已熟記／到期／錯題／全新 × 邊界值）；腳本 exit 0 且列印各案例 PASS。
- 已知不適用：涉及 Supabase 呼叫、DOM 渲染、TTS 的函式（需 Playwright 走真頁面）。
