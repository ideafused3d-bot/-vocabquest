執行收工反思，嚴格依 `docs/harness/F-KNOWLEDGE-PROTOCOL.md` §6 清單逐項做：

1. 本 session 有沒有 F §2 定義的「必須記」事件（耗 ≥2 輪派工的問題／未來會再踩的坑）？有 → 依 F §2 固定格式追加到 `docs/project/LESSONS.md`。
2. 有沒有解出「會重複使用」的固定模式？有 → 依 PLAYBOOK 檔頭格式（四欄齊全才收）追加到 `docs/project/PLAYBOOK.md`。
3. 本次改動有沒有讓 `docs/project/SPEC.md` 或 `docs/project/OPS.md` 過時？有 → 同步更新。
4. 有沒有想改但無權改的規則（FROZEN 檔、settings）？有 → 寫進 `docs/project/PROPOSALS.md`。
5. 檢查 F §4 門檻：LESSONS 是否超過 30 條／300 行？PROPOSALS 是否累積 ≥5 條未裁決？觸發了就照該節動作。
6. 最後向 User 一段話總結：做了什麼、記了什麼（列出新增的 L-XXX／PB-XXX 編號）、有哪些提案待裁決。
