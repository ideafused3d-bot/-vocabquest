1. 老師端操作 👩‍🏫

登入

開網站 → 「登入」分頁 → 輸入你的帳號與密碼。系統中第一個註冊的帳號就是老師，之後所有透過「建立老師帳號」分頁註冊的帳號都會是學生身分（防止學生自己升級成老師）。

新增學生帳號

  1. 底部導覽點「學生」→ 右上「＋ 新增學生」
  2. 填學生姓名、帳號（英數字，例如 ming01）、密碼（至少 6 碼）
  3. 把帳號密碼抄給學生。學生不需要 Email

💡 密碼建好後老師端看不到，請當下記下來。學生忘記密碼時的處理方式見第 6 章。

查看學生學習狀況

「學生」頁列表可看每位學生的連續天數 🔥、等級 ⭐、熟記數 ✅。點任一位學生進入詳情頁：三色進度條（已熟記／學習中／未學習）、被指派的單字集各自的熟記進度、以及「常答錯單字 Top 10」——下次上課針對這些單字加強最有效。

2. 單字要怎麼輸入 🔤

底部導覽點「單字集」→「＋ 新增單字集」，填名稱（例如 Unit 1 - 日常生活）、分類、難度（初級／中級／高級），然後輸入單字。有三種方式：

方式一：逐列輸入

直接在「英文」「中文」欄位打字，按「＋ 新增一列」繼續加。適合少量單字或臨時補字。

方式二：批次貼上（最快）

點單字清單右上角的「批次貼上」，把整理好的清單貼進去，一行一個單字，英文和中文用逗號隔開：

apple,蘋果
school,學校
beautiful,美麗的

按「加入清單」就會全部變成單字列。你平常用 Excel 或 Google 試算表整理單字的話，兩欄複製貼上就是這個格式。

方式三：從課本照片匯入 📷

網站目前不能直接上傳照片，但用手機內建的文字擷取功能，拍照 → 變文字 → 批次貼上，實際操作大約一分鐘：

  • iPhone：用相機拍下課本單字頁 → 打開照片 → 長按照片上的文字（或點右下角文字擷取圖示）→ 全選複製
  • Android：打開 Google 相簿或 Google 智慧鏡頭（Lens）→ 選照片 → 點「文字」→ 選取複製

貼到任何記事本整理成 英文,中文 的格式（每行一個），再貼進「批次貼上」即可。

💡 想要「上傳照片自動辨識單字」的一鍵功能？技術上可行（需串接 OCR / AI 服務），屬於未來擴充項目，跟開發者說一聲就能排進去。

發音從哪裡來？

不用準備音檔。學生按 🔊 時由瀏覽器內建的語音合成（Web Speech API）唸出英文，手機和電腦都支援。

3. 怎麼分別給學生出功課 📋

可以，而且這正是系統的核心設計。「指派單字集」就是出功課——每個單字集可以指派給不同的學生組合，每位學生只會看到（也只能練習）被指派給自己的單字集。

  1. 「單字集」頁 → 找到要出的單字集 → 按「指派」
  2. 勾選要練這份的學生（可只勾一位，也可勾多位）→「儲存指派」
  3. 取消勾選再儲存＝收回功課（學生的練習紀錄會保留）

實際運用範例：

情境做法
程度分班建「初級 Unit 1」指派給小明；建「高級 Unit 1」指派給小美
每週功課每週建一個新單字集（Week 1、Week 2…），週週指派
考前衝刺把段考範圍做成一個單字集，只指派給要考試的學生
補救教學看學生詳情頁的「常答錯 Top 10」，做成專屬複習集指派給他

💡 目前沒有「截止日期」功能，指派後學生隨時可練。系統的 SRS 排程（第 5 章）會自動安排每天該複習什麼，比死線更符合記憶科學。想加截止日提醒屬於可擴充項目。

4. 學生端操作 🧑‍🎓

用老師給的帳號密碼登入，看到自己的儀表板：連續學習天數 🔥、等級與 XP 進度條 ⭐、已熟記單字數 ✅、待複習數 ⏰。

  • 📅 每日複習：最重要的按鈕！系統自動排出今天到期的單字＋新單字，每天按一次持續累積
  • 單字集練習：想加強哪份功課，就按那份單字集的「練習」
  • 📕 錯題本：答錯 2 次以上的單字自動收錄，可針對性複習，熟記後自動移除
  • 題型：選擇題（看中文選英文）和拼字題（看中文＋聽發音，拼出英文）輪流出現；拼字題 XP 較高
  • 進度頁：看自己「已熟記／學習中／未學習」的比例和各單字集進度

🔥 連續天數的條件是「每天至少完成 1 題」，中斷就歸零重算——鼓勵學生每天打開練一下，比週末狂練一次有效得多。

5. 複習機制與遊戲化規則 🧠

系統採用間隔重複（SRS，Leitner 盒子法）：每個單字有 0～6 的熟練度等級，答對升一級、複習間隔拉長；答錯降兩級、隔天就要再複習。

熟練度0123456
下次複習當天1 天2 天4 天7 天15 天30 天

熟練度達 5 以上＝「已熟記」。之後仍會偶爾出現複習，防止遺忘。

XP 與等級

  • 選擇題答對 +10 XP、拼字題答對 +15 XP、答錯也有 +2 XP 安慰分
  • 升到下一級需要「目前等級 × 100」XP（Lv.1→2 要 100，Lv.2→3 要 200…）
  • XP 由資料庫端計算，學生沒辦法用瀏覽器小手段作弊改分

6. 疑難排解 🔧

登入不了，怎麼判斷原因？

畫面顯示原因解法
「帳號或密碼錯誤」 打錯字、大小寫、或帳號根本沒建成／被刪除 帳號一律小寫；到 Supabase → Authentication → Users 確認帳號存在，不存在就重建
「帳號存在但缺少資料檔」 帳號比資料庫規則先建立（缺 profile） SQL Editor 執行 repair-profiles.sql
登入後馬上被彈回登入頁 舊版網頁快取 重新整理（手機：關掉分頁重開；電腦：Ctrl+Shift+R）

還是不行？在 Supabase SQL Editor 執行 diagnose.sql，會列出所有帳號的狀態（有沒有資料檔、角色對不對、有沒有登入過），把結果貼給協助你的人一看就知道。

老師帳號忘記密碼／被誤刪

  1. 到網站「建立老師帳號」分頁，註冊一個新帳號（它會先是學生身分）
  2. Supabase SQL Editor 執行：update public.profiles set role = 'teacher' where username = '新帳號名';
  3. 重新登入，就是老師了。舊資料（單字集、學生）都還在

學生忘記密碼

目前無法在網站上重設。最快的做法：老師在 Supabase Dashboard → Authentication → Users 刪掉該帳號，回老師後台重建一個（帳號名可以一樣），再重新指派單字集。注意：練習進度會歸零。

其他常見問題

  • 學生登入後看不到任何單字集？老師還沒指派。到「單字集」→「指派」勾選該學生。
  • 按 🔊 沒聲音？確認手機沒開靜音、媒體音量有開；少數舊瀏覽器不支援語音合成，換 Chrome 或 Safari 試試。
  • 改了單字內容，學生的進度會消失嗎？編輯既有單字（改拼字或中文）會保留進度；「刪掉單字」才會連進度一起刪。
  • 要收費嗎？Supabase 免費額度對家教規模（數十位學生）綽綽有餘。
